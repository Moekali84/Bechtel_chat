## 1. Introduction

PBIChat ("we", "us", "our") provides an AI-powered chat assistant for Microsoft Power BI that enables natural-language queries against your databases. This Privacy Policy explains what data we collect, how it flows through our systems, and your rights regarding that data.

This policy applies to:
- The PBIChat Power BI custom visual
- The PBIChat backend API
- The PBIChat website (https://pbichat.com)

## 2. Data We Collect

### 2.1 Usage Data
- Chat messages you send to the assistant
- SQL queries generated by the AI and executed against your databases
- Query results returned from your databases (processed in memory, not persisted)
- Daily query counts per user or per IP address (for free-tier enforcement)
- Timestamps of interactions

### 2.2 Technical Data
- IP address (used for rate limiting and anonymous usage tracking)
- HTTP request metadata (User-Agent, request timestamps)

### 2.3 Payment Data
- Managed entirely by Microsoft through AppSource. Billing, subscriptions, and payment processing are handled by Microsoft's commerce platform. We do not store credit card numbers, CVVs, or billing addresses.

### 2.4 Data We Do NOT Collect
- Power BI report data beyond what you explicitly query through the chat
- Telemetry or analytics tracking (no tracking pixels, no behavioral analytics)
- Cookies for tracking purposes (see our [Cookie Policy](cookie-policy.md))

## 3. How Data Flows

```
Power BI (Visual)
  → PBIChat Backend (FastAPI)
      → Azure OpenAI → LLM (GPT) — processes your question, generates SQL
      → Your Database (Databricks / SQL Server) — executes generated SQL
      → Microsoft AppSource — handles billing and subscriptions (Pro tier)
  ← Response returned to Visual
```

1. **Visual → Backend:** Your chat message is sent over HTTPS to the PBIChat backend.
2. **Backend → Azure OpenAI:** Your message (plus database schema context) is sent to Azure OpenAI to generate SQL queries.
3. **Backend → Your Database:** Generated SQL queries are executed against your own database connections (Databricks or SQL Server). Query results stay in memory for the duration of the request and are not persisted.
4. **Backend → Microsoft AppSource:** Payment processing for Pro subscriptions is handled by Microsoft through AppSource.
5. **Backend → Visual:** The final AI-generated response is returned to the Power BI visual over HTTPS.

## 8. Your Rights

### 8.1 GDPR Rights (EEA/UK Residents)
You have the right to:
- **Access** your personal data — request a copy of all data we hold about you
- **Rectification** — correct inaccurate personal data
- **Erasure** ("right to be forgotten") — request deletion of your account and associated data
- **Restrict processing** — limit how we use your data
- **Data portability** — receive your data in a structured, machine-readable format
- **Object** — object to processing based on legitimate interest
- **Withdraw consent** — where processing is based on consent

### 8.2 CCPA Rights (California Residents)
You have the right to:
- **Know** what personal information we collect and how it is used
- **Delete** your personal information
- **Opt-out of sale** — we do not sell personal information
- **Non-discrimination** — we will not discriminate against you for exercising your rights

### 8.3 Exercising Your Rights
To exercise any of these rights, contact us at **support@pbichat.com**. We will respond within 30 days (GDPR) or 45 days (CCPA).

## 9. Data Security

- All data in transit is encrypted via HTTPS/TLS
- Database credentials and API keys are stored as environment variables, never in code
- Settings access is protected by a server-side password
- Rate limiting protects against abuse
- SQL injection prevention blocks destructive queries
- Error messages are scrubbed of sensitive data (API keys, credentials)

## 10. International Data Transfers

Data may be transferred to and processed in the United States (where Microsoft Azure operates). For EEA/UK users, these transfers are protected by:
- Standard Contractual Clauses (SCCs) with our sub-processors
- Microsoft's compliance with applicable data protection frameworks
